# CyberCycho — Visual Identity Guide

Retro-pixel / synthwave-cyberpunk identity. One system, used everywhere:
this website, itch.io/Steam pages, and in-game (splash screens, app icons,
loading screens).

## Logo

- `icon-mark.svg` / `icon-mark-1024.png` — icon mark, dark rounded-square
  backdrop. Use as app/game icon, favicon source, social avatar.
- `icon-mark-transparent.svg` / `.png` — icon only, no backdrop. Use when
  compositing over your own background (loading screens, watermarks).
- `icon-mark-mono.svg` — single-color (no glitch offset), for contexts that
  need one flat color (embroidery, laser-etch, single-color UI icon slots).
- `wordmark-transparent.png` / `wordmark-on-dark.png` — "CyberCycho"
  logotype, Press Start 2P, hard pixel edges, on transparent or dark bg.
- `logo-horizontal.png` — icon + wordmark side by side. Primary lockup for
  wide spaces (website header, Steam capsule, YouTube banner).
- `logo-stacked.png` — icon above wordmark. Use for square/portrait spaces
  (social avatar with text, splash screen, itch.io cover).

**Clear space:** keep empty space around the logo equal to at least the
height of the icon's rounded-square backdrop. Don't crowd it against edges
or other text.

**Minimum size:** icon mark readable down to 16px (it's designed on a 16×16
pixel grid specifically for this). Wordmark: don't go below ~120px wide —
Press Start 2P stops reading as text below that.

**Don't:**
- Don't recolor the icon/wordmark outside the palette below.
- Don't apply smoothing/anti-aliasing filters — every edge must stay a hard
  pixel step. Always scale with nearest-neighbor, never bilinear/bicubic.
- Don't stretch non-uniformly. Don't rotate.
- Don't remove the pink/yellow glitch offset from the primary lockups — it's
  part of the mark, not a rendering effect layered on top.

## Color palette

| Name | Hex | Use |
|---|---|---|
| BG (background) | `#0a0a12` | page/scene background, icon backdrop |
| Card | `#14141f` | panels, cards, UI containers |
| Border | `#2a2a3d` | hairline borders, dividers |
| Neon Cyan | `#2effe0` | primary brand color — logo fill, links, headers |
| Neon Pink | `#ff2e88` | glitch offset (left), accents, tags |
| Neon Yellow | `#f4e04d` | glitch offset (right), highlights, dates/labels |

The "glitch" effect is always the same recipe: cyan main layer, pink offset
~1 unit left at ~80% opacity, yellow offset ~1 unit right at ~63% opacity.
Same recipe scales from a 16px favicon to a 1024px splash — offsets are
defined in grid units, not fixed pixels, so they scale with the asset.

## Typography

- **Display / headings / logo:** Press Start 2P (`fonts/PressStart2P-Regular.ttf`).
  Bitmap-style, all-caps reads best. Use sparingly — titles, section headers,
  the logo. It's wide and slow to read in long strings.
- **Body / UI text:** VT323 (`fonts/VT323-Regular.ttf`). Monospace terminal
  feel, reads fine at paragraph length. Use for descriptions, dates, body copy.
- Both are OFL-licensed — free to embed in a game build, no attribution
  requirement (though crediting the type authors in credits is good practice).

## Using this in a game engine

- **Icons:** `icon-*.png` covers the common target sizes (16/32/48/180/192/
  512/1024) — map to whatever your engine's icon import step asks for.
  `favicon.ico` is a proper multi-resolution ICO (16/32/48/256) for Windows
  builds / web favicon.
- **Fonts:** copy `fonts/*.ttf` into your project's font assets and import
  like any other font file — no network dependency, MIT/OFL-equivalent to
  redistribute.
- **Storefront art:** `logo-horizontal.png` fits Steam capsule proportions
  reasonably; `logo-stacked.png` fits itch.io's square/portrait cover slot.
  Composite onto a `#0a0a12` background for either if you need an opaque
  (non-transparent) submission.
- **Vector source:** the `.svg` icon files are hand-built from the same
  16×16 pixel grid as the PNGs (see `icon-mark.svg`) — edit the grid in
  `build_brand.py` if the mark itself ever needs to change, then regenerate
  everything from that one source of truth rather than hand-editing exports.

## Source of truth

All assets here are generated by `scripts/build_brand.py` (Pillow) from a
single pixel-grid definition + the palette constants — regenerate rather
than hand-edit exported PNGs/SVGs if the mark changes.
